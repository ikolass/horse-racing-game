### Race Track Internal Layout

```
[ Round Info Header — full column width, 48px tall ]
[ Lane 1: Label (120px) | Track Area (remaining width) ]
[ Lane 2: Label (120px) | Track Area (remaining width) ]
...
[ Lane 10: Label (120px) | Track Area (remaining width) ]
```

- Track area total height: 10 lanes × 40px = 400px (source: CONTEXT.md D-07)
- Label column: fixed 120px wide (source: CONTEXT.md D-08)
- Track column: `calc(100% - 120px)` — marker travels here
- Finish line: positioned at `right: 0` of the track column, full lane height
- Component minimum width: inherits center column (55% of 1024px = ~563px)

## Color

Source: `src/assets/base.css` — tokens established in Phase 2. No new color values.

| Role | Value | Usage in Phase 3 |
|------|-------|------------------|
| Dominant (60%) | `#1a1a2e` (`--color-bg-dominant`) | Even-numbered lane backgrounds (1, 3, 5, 7, 9) |
| Secondary (30%) | `#16213e` (`--color-bg-secondary`) | Odd-numbered lane backgrounds (2, 4, 6, 8, 10); round info header background |
| Accent (10%) | `#e94560` (`--color-accent`) | Finish line only |
| Destructive | — | No destructive actions in Phase 3 |

Accent (`#e94560`) is reserved for:
- Finish line: 2px solid vertical rule at the right edge of each lane's track area

Text colors (inherited from Phase 2 — same tokens):
- Primary text: `#e0e0e0` (`--color-text-primary`) — horse names in lane labels and moving markers
- Muted text: `#6b7280` (`--color-text-muted`) — idle/pre-race state instructional text
- Badge background: `#0f3460` (`--color-badge-bg`) — condition score badge on moving marker
- Badge text: `#e0e0e0` (`--color-text-primary`)

Lane dividers: 1px solid `#2d3748` (`--color-divider`) — bottom border on each lane (source: CONTEXT.md D-07).

Horse color swatches on moving markers: each horse's own `horse.color` hex value from Vuex state
(inline style `background-color: horse.color` — 20 distinct colors pre-baked in Phase 1).

### Discretion Decisions (Claude's)

**Finish line color:** `var(--color-accent)` (`#e94560`), 2px solid. Accent is already reserved
for landmark interactive elements; the finish line is a landmark spatial element that justifies the
same treatment. It is visually distinct from the dominant/secondary lane backgrounds without
requiring a new color token.

**Lane alternation:** Even-numbered lanes use `--color-bg-dominant`, odd-numbered lanes use
`--color-bg-secondary`. The 10-lane grid is dense (400px for 10 rows); alternating backgrounds
create visual row separation without adding explicit horizontal rules that would compete with the
lane divider borders.

**Round header background:** `--color-bg-secondary` (`#16213e`) — one step darker than the
dominant background, visually separates the header from the lane area without a hard border.

## Interaction Contract

### Pre-Race State (gamePhase: IDLE or SCHEDULED)

- All 10 horses for the upcoming round displayed in starting positions (marker at `left: 0%`)
- Lane labels visible with horse name and color swatch
- Moving markers visible at start line (condition badge shown)
- Round info header shows "Round 1 — 1200m" (first scheduled round)
- Finish line visible at right edge of track area
- No animation running; no tick loop active

Source: CONTEXT.md `<specifics>` — "During idle/scheduled state the track shows 10 horses
at position 0, so the layout is visible before animation starts."

### Racing State (gamePhase: RACING)

- Tick fires every 50ms; `SET_POSITIONS` committed each tick
- Each horse marker moves smoothly left-to-right via `transition: left 45ms linear`
- Speed per tick: `horse.condition / (maxConditionInRound * 60)` — deterministic, no per-tick randomness (source: CONTEXT.md D-03, RESEARCH.md Pattern 1)
- Higher-condition horses visibly outpace lower-condition horses (RACE-02)
- Winner (highest-condition horse in round) reaches finish in ~3 seconds (60 ticks × 50ms) (source: CONTEXT.md D-02)
- Slowest horse arrives ~5–6 seconds (condition-proportional)
- Round info header shows current round number and distance (ANIM-03)
- Lane labels remain static throughout (ANIM-02)
- Condition score badge remains visible on each moving marker (ROST-03)

### Between-Round Pause (gamePhase: ROUND_COMPLETE)

- All 10 horse markers frozen at `left: 100%` (clamped at finish line)
- Pause duration: 1500ms
- Round header and lane labels remain visible — no overlay, no banner (source: CONTEXT.md D-12)
- No user interaction required or possible during pause

### Auto-Advance (gamePhase: RACING, next round)

- After 1500ms pause: `RESET_POSITIONS` → all markers return to `left: 0%`
- Round info header updates in place to show next round (e.g., "Round 3 — 1600m")
- No blank flash between rounds — header updates synchronously before tick restarts
- New tick loop starts immediately (source: CONTEXT.md D-13)

Source for "in-place update" (Claude's discretion): Blanking the track between rounds would
create a jarring flash for 10-lane content. In-place update is smoother and keeps the user
oriented.

### Final State (gamePhase: DONE)

- All 10 horses from round 6 frozen at final positions
- Round info header shows "Round 6 — 2200m"
- No overlay, no "Race complete" banner — Results panel (Phase 4) handles completion messaging (source: CONTEXT.md D-14)
- Track remains as last-round snapshot until user clicks Generate again

### CSS Transition Spec

```css
.horse-marker {
  position: absolute;
  top: 50%;
  transform: translateY(-50%);
  transition: left 45ms linear;  /* 45ms < 50ms tick — prevents stacked transitions */
  display: flex;
  align-items: center;
  gap: var(--space-xs);
  white-space: nowrap;
}
```

Source: RESEARCH.md Pattern 3. `transition: left` only — not `transition: all`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Round info header | "Round {N} — {distance}m" (e.g., "Round 2 — 1400m") |
| Pre-race horse count | Not shown — lanes speak for themselves |
| Empty state heading | "Waiting for schedule…" |
| Empty state body | "Click Generate Schedule to set up a race." |
| Error state | No error states in Phase 3 (Vuex action errors are console-only; no UI error display required) |
| Destructive confirmation | No destructive actions in Phase 3 |
| Done state | No completion copy — track shows last round snapshot; Results panel handles messaging |

Source: CONTEXT.md D-10 for round header format. CONTEXT.md D-14 for done state (no banner).

Empty state applies to: center column when `gamePhase === 'IDLE'` and no round data is
available (schedule not yet generated). Once schedule exists (`gamePhase === 'SCHEDULED'`),
round 1 horses are shown at position 0 — no empty state copy needed.
